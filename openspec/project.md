# Project Context

## Purpose
The project aims to integrate Adversarial Autoencoders (AAE) with machine learning methods to perform objective classification of financial transactions. The goal represents a fraud detection system that classifies transactions into three types:
- Regular
- Local Anomaly
- Global Anomaly

The core approach involves training an AAE to generate latent vectors, which are then used as features for supervised learning algorithms (SVM, Random Forest, Naive Bayes) to improve classification performance compared to using original features.

## Tech Stack
- **Languages**: Python 3
- **Deep Learning Framework**: PyTorch (`torch`, `torch.nn`)
- **Machine Learning**: Scikit-learn (`sklearn`)
    - Classifiers: SVM, Random Forest, GaussianNB
    - Metrics: ROC, MAUC, Precision-Recall
    - Model Selection: StratifiedKFold
- **Data Manipulation**: Pandas, NumPy
- **Visualization**: Matplotlib, Seaborn

## Project Conventions

### Code Style
- **Naming**: Mixed use of `camelCase` (e.g., `latentVecDim`, `datasetFile`) and `snake_case` (e.g., `seed_value`, `clf_name`). Class names use `PascalCase`.
- **Formatting**: 4-space indentation.
- **Comments**: Generous use of comments to denote sections (e.g., `importing pytorch libraries`, `specify first layer`).

### Architecture Patterns
- **Modular Components**: Network components are separated into individual files (e.g., `Encoder.py`, `Decoder.py`, `Discriminator.py`).
- **Experiment Workflow**: 
    1.  **Train**: `validation_v0.py` trains the autoencoder.
    2.  **Generate**: `generateDatasets.py` creates latent datasets in `latent_data_sets/`.
    3.  **Experiment**: `MAUC_Exp.py` (latent features) and `MAUC_rawExp.py` (original features) run classification experiments.
    4.  **Results**: Outputs are saved to `results/`.

### Directory Structure
- `data/`: Input datasets.
- `latent_data_sets/`: Generated latent vector datasets.
- `models/`: Saved model artifacts.
- `results/`: Experiment outputs (plots, text files).
- `openspec/`: Project specifications and agent instructions.

### Testing Strategy
- **Ad-hoc Testing**: Testing appears to be done via specific test scripts (e.g., `test_gaussianmixplot.py`, `testingBarPlot.py`) rather than a comprehensive unit test suite.
- **Validation**: `validation_v0.py` serves as the primary validation script for the autoencoder training.

## Domain Context
- **Fraud Detection**: The domain focuses on identifying anomalies in financial data.
- **MAUC**: Metric Area Under the Curve, a key performance metric used in this project.
- **Latent Space**: The compressed representation of data generated by the autoencoder, critical for feature engineering in this pipeline.

## External Dependencies
- **PyTorch**: Core dependency for deep learning.
- **Scikit-learn**: Essential for traditional ML classifiers and evaluation metrics.
